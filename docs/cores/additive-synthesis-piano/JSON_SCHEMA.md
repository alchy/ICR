# ICR -- JSON Soundbank Schema

Reference for all keys in the AdditiveSynthesisPianoCore soundbank JSON.
Generated by the exporter (`training/modules/exporter.py`), loaded by
`additive_synthesis_piano_core.cpp::load()` and `loadBankJson()`.

---

## File Structure

```json
{
  "metadata": {
    "instrument_name": "pl-grand",
    "midi_range_from": 21,
    "midi_range_to":   108,
    "source":          "soundbank:params",
    "sr":              48000,
    "target_rms":      0.06,
    "vel_gamma":       0.7,
    "k_max":           60,
    "rng_seed":        0,
    "duration_s":      3.0
  },
  "notes": {
    "m060_vel3": { ... },
    ...
  }
}
```

---

## 1. Bank-level Keys

### 1a. Top-level

| Key | Type | Description | C++ |
|---|---|---|---|
| `metadata` | object | Bank metadata -- see 1b | ignored |
| `notes` | object | Note dictionary, key = `"m{midi:03d}_vel{vel}"` | loaded |

### 1b. `metadata` Object

| Key | Type | Description |
|---|---|---|
| `instrument_name` | string | Instrument name (user-defined) |
| `midi_range_from` | int | Lowest MIDI note with measured samples |
| `midi_range_to` | int | Highest MIDI note with measured samples |
| `source` | string | `"soundbank:params"` / `"nn-hybrid:model"` / `"dna"` |
| `sr` | int | Sample rate (44100 or 48000) |
| `target_rms` | float | Target RMS level (baked into `rms_gain`) |
| `vel_gamma` | float | Velocity gamma curve (default 0.7) |
| `k_max` | int | Max partial count (60) |
| `rng_seed` | int | Seed for phase generation (phi, phi_diff) |
| `duration_s` | float | Duration of rendered notes in seconds |

## 2. Note-level Keys

### 2a. Identification

| Key | Dep | JSON | C++ load | C++ synth | SysEx | Description |
|---|---|---|---|---|---|---|
| `midi` | -- | required | meta | meta | -- | MIDI note 21-108 |
| `vel` | -- | required | meta | meta | -- | Velocity index 0-7 |
| `is_interpolated` | m,v | optional | `value("is_interpolated", false)` | -- | -- | `true` = NN-generated note |

### 2b. Synthesis Parameters

| Key | Dep | Default | C++ load | C++ use | SysEx | Description |
|---|---|---|---|---|---|---|
| `f0_hz` | m | 440.0 | required | oscillators, inharmonicity | yes | Fundamental frequency (Hz) |
| `B` | m | 0.0 | `value("B", 0.0)` | `f_k = k*f0*sqrt(1+B*k^2)` | yes | Inharmonicity coefficient |
| `attack_tau` | m,v | 0.05 | required | noise env decay | yes | Noise decay tau (s) |
| `A_noise` | m,v | 0.04 | required | noise amplitude | yes | Noise amplitude (0..1) |
| `noise_centroid_hz` | m,v | 3000 | `value("noise_centroid_hz", 3000)` | biquad BPF center | yes | Noise bandpass center (Hz, Q=1.5) |
| `rms_gain` | m,v | 1.0 | required | per-note gain | yes | RMS calibration gain |
| `phi_diff` | m,v | 0.0 | required | string 2/3 phase offset | yes | Phase offset between strings (rad) |
| `stereo_width` | m,v | 1.0 | `value("stereo_width", 1.0)` | M/S post-EQ: `S *= width` | yes | Stereo width correction factor |
| `rise_tau` | m | -1 | `value("rise_tau", -1.0)` | attack rise env | -- | Rise time (s). -1 = midi-based default |
| `n_strings` | m | -1 | `value("n_strings", -1)` | 1/2/3-string model | -- | -1 = midi-based default (<=27->1, <=48->2, >48->3) |
| `decor_strength` | m | -1 | `value("decor_strength", -1.0)` | allpass decorrelation | -- | -1 = midi-based default |

**Runtime override:** At noteOn, `rms_gain` is always taken from the forte
(highest valid) velocity layer for the same MIDI note. The per-layer `rms_gain`
in JSON is ignored in favor of the forte reference. This ensures consistent
loudness normalization since forte has the best SNR.

Similarly, spectral shape (A0 ratios for k>1 relative to k=1) is normalized
to match the forte layer. See OVERVIEW.md "Velocity Model" for details.

### 2c. EQ Biquad Cascade (`eq_biquads`)

Optional array of biquad sections. If absent, no spectral EQ is applied.
Max 10 sections (`PIANO_N_BIQUAD`).

```json
"eq_biquads": [
  { "b": [b0, b1, b2], "a": [a1, a2] },
  { "b": [b0, b1, b2], "a": [a1, a2] },
  ...
]
```

- `b0, b1, b2` -- numerator coefficients (a0 = 1 implicit)
- `a1, a2` -- denominator coefficients

Cascade of 5 sections = 10th-order min-phase IIR filter.

C++ loads into `PianoNoteParam::eq[PIANO_N_BIQUAD]` and applies via
`eq_cascade_stereo()` with a -3 dB gain floor.

### 2d. Spectral EQ Curve (`spectral_eq`)

Raw frequency/gain curve from the EQ fitter. Stored so the editor can
re-fit biquads after curve edits. **Not used by C++ runtime** -- only
`eq_biquads` is loaded.

```json
"spectral_eq": {
  "freqs_hz":           [80.0, 90.2, ..., 18000.0],
  "gains_db":           [-2.1, -1.8, ..., 0.5],
  "stereo_width_factor": 1.23
}
```

| Key | Type | Description |
|---|---|---|
| `freqs_hz` | float[] | 64 log-spaced frequencies (80 Hz - 18 kHz) |
| `gains_db` | float[] | EQ gain at each frequency (dB, 0 dB = mean) |
| `stereo_width_factor` | float | S/M ratio: original vs synthesized. Exported as top-level `stereo_width`. Clamped [0.2, 2.0] |

### 2e. Diagnostic Keys (GUI-only, no RT impact)

| Key | Default | Description |
|---|---|---|
| `fit_quality` | 0.0 | Bi-exp fit quality (0..1, per-partial). GUI color-coded. |
| `damping_derived` | false | True if tau1 was replaced by damping law R+eta*f^2 |
| `is_interpolated` | false | True if NN-generated note. GUI shows [NN]/[MEASURED] badge |

## 3. Partial-level Keys

Each note has a `partials` array -- max 60 objects (`PIANO_MAX_PARTIALS`).

| Key | Default | C++ use | SysEx | Description |
|---|---|---|---|---|
| `k` | -- | partial index | -- | 1-based partial index |
| `f_hz` | -- | oscillator frequency | `0x10` | Frequency (Hz). Recomputed when B changes. |
| `A0` | -- | amplitude | `0x11` | Initial amplitude |
| `tau1` | 0.5 | env fast decay | `0x12` | Fast tau of bi-exp (s). Floor 0.05 in exporter. |
| `tau2` | =tau1 | env slow decay | `0x13` | Slow tau (s). Always >= tau1. |
| `a1` | 1.0 | env mix weight | `0x14` | Fast component weight. 1.0 = mono-exponential. |
| `beat_hz` | 0.0 | string detuning | `0x15` | Beat frequency (Hz). 0 = single string. |
| `phi` | 0.0 | initial phase | `0x16` | Phase of string 1 (rad) |
| `fit_quality` | 0.0 | (GUI only) | -- | Fit quality (0..1) |
| `damping_derived` | false | (GUI only) | -- | tau1 from damping law |

---

## 4. Keys NOT in Soundbank JSON

These exist in extractor output but the exporter does not write them:

| Key | Level | Why absent |
|---|---|---|
| `beat_depth` | partial | C++ does not implement AM beating |
| `mono` | partial | Exporter sets `beat_hz=0.0` instead |
| `is_longitudinal` | partial | C++ has no longitudinal partial model |

## 5. Synthesis Model

```
partial[k]:
  env(t) = a1*exp(-t/tau1) + (1-a1)*exp(-t/tau2)   # bi-exp; a1=1 -> mono-exp

  # 1-string (MIDI <= 27, bass)
  out(t) = A0*env(t)*cos(2pi*f_hz*t + phi)

  # 2-string (MIDI 28-48, tenor)
  s1(t) = cos(2pi*(f_hz + beat_hz/2)*t + phi)
  s2(t) = cos(2pi*(f_hz - beat_hz/2)*t + phi + phi_diff)
  out(t) = A0*env(t)*(s1 + s2) / 2

  # 3-string (MIDI > 48, treble, symmetric)
  s1(t) = cos(2pi*(f_hz - beat_hz)*t + phi)           # outer left
  s2(t) = cos(2pi*f_hz*t + phi2)                      # center (phi2 random per noteOn)
  s3(t) = cos(2pi*(f_hz + beat_hz)*t + phi + phi_diff) # outer right
  out(t) = A0*env(t)*(s1 + s2 + s3) / 3

noise(t):
  white(t) = randn() * A_noise * exp(-t/attack_tau)
  bpf = rbj_bandpass(noise_centroid_hz, Q=1.5, sr)
  y(t) = biquad_tick(white(t), bpf)                  # L, R independent

note(t) = rms_gain * vel_gain * [sum(partial[k](t)) * rise_env + noise(t)]
        -> biquad EQ cascade (eq_biquads, -3 dB gain floor)
        -> M/S: M=(L+R)/2  S=(L-R)/2*stereo_width  L=M+S  R=M-S

vel_gain = pow(vel_norm, 1.5)    # vel_norm = midiVelToFloat(velocity) / 7.0
```

**rms_gain** is calibrated over the mono M channel (= (L+R)/2) -- invariant
to M/S operation, so it applies without recalculation after stereo_width.

**beat_hz semantics**: 2-string model uses beat_hz/2 per side (total beat_hz);
3-string model uses beat_hz per side (2x frequency spread, 2 amplitude nulls
per beat period).
