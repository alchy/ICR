# PhysicalModelingPianoCore -- JSON Soundbank Schema

Per-note physical parameters for the digital waveguide piano model.
Generated by `tools/generate_physical_bank.py`, loaded by `PhysicalModelingPianoCore::load()`.

---

## File structure

```json
{
  "metadata": {
    "instrument_name": "steel-string-piano",
    "version": 1,
    "sr": 48000,
    "model": "PhysicalModelingPianoCore",
    "created": "2026-04-08T13:05:00",
    "description": "..."
  },
  "notes": {
    "m021": { ... },
    "m022": { ... },
    ...
    "m108": { ... }
  }
}
```

Note key format: `m{midi:03d}` (no velocity layers -- velocity controls
hammer speed directly in the physics model).

---

## Metadata

| Key | Type | Description |
|-----|------|-------------|
| `instrument_name` | string | Display name |
| `version` | int | Schema version (currently 1) |
| `sr` | int | Sample rate (48000) |
| `model` | string | Target core: `"PhysicalModelingPianoCore"` |
| `created` | string | ISO timestamp |
| `description` | string | Free-form notes |

## Per-note parameters

### String physical properties

| Key | Type | Range | Default | Description |
|-----|------|-------|---------|-------------|
| `midi` | int | 21-108 | -- | MIDI note number |
| `f0_hz` | float | 27-4200 | from MIDI | Fundamental frequency (Hz) |
| `B` | float | 0-0.01 | MIDI-based | Inharmonicity coefficient. Higher = stiffer string = more stretched partials. Bass wound strings: 1-2e-3, treble: 1-5e-5. `f_k = k * f0 * sqrt(1 + B * k^2)` |
| `gauge` | float | 0.5-5.0 | MIDI-based | String thickness (relative). Affects: excitation spectrum (thicker = stronger fundamental), loss filter (thicker = more HF damping). Bass: 3-4, middle: 2, treble: 1 |
| `T60_fund` | float | 0.5-20 | MIDI-based | T60 decay time of fundamental (seconds). Time for the fundamental to drop 60 dB. Bass: 10-12s, treble: 1-2s |
| `T60_nyq` | float | 0.01-1.0 | MIDI-based | T60 at Nyquist frequency (seconds). Controls spectral tilt -- how much faster high harmonics decay vs fundamental. Shorter = warmer, longer = brighter. Effective value scaled by gauge: `T60_nyq_eff = T60_nyq / gauge` |
| `n_strings` | int | 1-3 | MIDI-based | Unison strings. MIDI <= 27: 1, MIDI 28-48: 2, MIDI > 48: 3 |
| `detune_cents` | float | 0-5 | MIDI-based | Detuning between unison strings (cents). Creates beating. Bass: 2-3, treble: 0.3-0.5 |

### Excitation (hammer/pluck) parameters

| Key | Type | Range | Default | Description |
|-----|------|-------|---------|-------------|
| `exc_rolloff` | float | 0-2.0 | 0.1 | Harmonic amplitude rolloff: `A_k ~ 1/k^rolloff`. 0 = flat (all harmonics equal), 2.0 = triangle (nylon-like). Steel piano: 0.1-0.3 |
| `exc_x0` | float | 0.05-0.25 | 0.143 (1/7) | Strike position as fraction of string length. Creates spectral notches at `k = 1/x0` multiples (k=7,14,21 for x0=1/7). Standard piano hammer: ~1/8 |
| `odd_boost` | float | 1.0-3.0 | 1.8 | Amplitude multiplier for odd harmonics (k=1,3,5,7...). Gives metallic/bell character. Physical basis: rigid termination + bridge coupling prefer odd modes. Bass: 2.0, treble: 1.5 |
| `knee_k` | int | 5-30 | 10 | Two-stage excitation knee. Harmonics below knee have mild rolloff (rich body), above knee have steep rolloff (smooth waveform). Higher knee = more body harmonics |
| `knee_slope` | float | 1.0-5.0 | 3.5-4.0 | Rolloff exponent above knee. `A_k ~ (knee_k/k)^slope` for k > knee_k. Higher = smoother waveform, lower = more brightness |
| `n_harmonics` | int | 20-80 | 80 | Number of harmonics in initial excitation. More = denser waveform. Limited by Nyquist: `k_max = sr / (2 * f0)` |

### Dispersion (inharmonicity in propagation)

| Key | Type | Range | Default | Description |
|-----|------|-------|---------|-------------|
| `n_disp_stages` | int | 0-16 | from B | Number of first-order allpass filters in the waveguide loop. Each stage adds frequency-dependent phase shift that stretches upper partials. More stages = more inharmonicity. Auto-computed: `min(16, int(B * N^2 * 0.5))` |
| `disp_coeff` | float | -0.3-0 | -0.15 | Per-stage allpass coefficient. Must be small (> -0.3) to avoid buzz artifacts. Single strong allpass creates distortion; cascade of mild allpass is stable |

## Loss filter (derived, not stored)

The one-pole loss filter coefficients are computed at runtime from
`T60_fund`, `T60_nyq`, and `gauge`:

```
N = sr / f0                          (samples per round-trip)
T60_nyq_eff = T60_nyq / gauge       (gauge scales HF damping)
g_dc  = 10^(-3 * N / (T60_fund * sr))   (gain at DC per loop)
g_nyq = 10^(-3 * N / (T60_nyq_eff * sr)) (gain at Nyquist per loop)
pole  = (g_dc - g_nyq) / (g_dc + g_nyq)

H(z) = g_dc * (1 - pole) / (1 - pole * z^-1)
```

Typical values:
- Bass (MIDI 36): g_dc=0.988, g_nyq=0.35, pole=0.48
- Middle (MIDI 60): g_dc=0.996, g_nyq=0.82, pole=0.10
- Treble (MIDI 84): g_dc=0.998, g_nyq=0.96, pole=0.02

---

## Excitation spectrum shape

The initial waveform is computed as a Fourier series:

```
delay[i] = sum over k=1..n_harmonics of:
    A_k * sin(2 * pi * f_k * i / N)

where:
    modal_k = sin(k * pi * exc_x0)           (strike position coupling)
    f_k = k * sqrt(1 + B * k^2)              (inharmonicity)

    if k <= knee_k:
        A_k = modal_k / k^exc_rolloff        (mild rolloff)
    else:
        A_k = A_knee * (knee_k / k)^knee_slope  (steep drop)

    if k is odd:  A_k *= odd_boost
    if gauge != 1: A_k *= gauge^(1 - k*0.05)  (thicker = stronger fundamental)
```

## Comparison with AdditiveSynthesisPianoCore JSON

| Aspect | Additive | Physical |
|--------|----------|----------|
| Notes | `m060_vel3` (per-velocity) | `m060` (velocity = hammer speed) |
| Partials | Explicit `partials[]` array | Generated from excitation params |
| Decay | `tau1`, `tau2`, `a1` per partial | `T60_fund`, `T60_nyq` (filter-based) |
| Spectral EQ | `eq_biquads` (10-section IIR) | Loss filter + soundboard IR |
| Noise | `A_noise`, `noise_centroid_hz` | Implicit from excitation |
| Inharmonicity | `B` + pre-computed `f_hz` | `B` + dispersion allpass cascade |
| Beating | `beat_hz` per partial | `detune_cents` between strings |
| Velocity layers | 8 layers (vel 0-7) | Continuous (hammer velocity) |
