# PhysicalModelingPianoCore -- JSON Soundbank Schema (v2)

Per-note physical parameters for the dual-rail waveguide piano model
with Chaigne-Askenfelt hammer excitation.

Generated by `tools-physical/generate_teng_v2.py`, loaded by
`PhysicalModelingPianoCore::load()` and `loadBankJson()`.

---

## File structure

```json
{
  "metadata": {
    "instrument_name": "steel-string-piano",
    "version": 2,
    "sr": 48000,
    "model": "teng-dual-rail-v2",
    "created": "2026-04-09T08:00:00",
    "description": "Dual-rail waveguide with Chaigne-Askenfelt hammer"
  },
  "notes": {
    "m021": { ... },
    "m022": { ... },
    ...
    "m108": { ... }
  }
}
```

Note key format: `m{midi:03d}` (no velocity layers -- velocity controls
hammer speed directly in the Chaigne physics model).

| Key | Type | Description |

## Per-note parameters

### String physical properties

| Key | Type | Range | Default | Description |
|-----|------|-------|---------|-------------|
| `midi` | int | 21-108 | -- | MIDI note number |
| `f0_hz` | float | 27-4200 | from MIDI | Fundamental frequency (Hz) |
| `B` | float | 0-0.01 | MIDI-based | Inharmonicity coefficient. `f_k = k * f0 * sqrt(1 + B * k^2)`. Bass wound: 1-2e-3, treble: 1-5e-5 |
| `gauge` | float | 0.8-3.0 | MIDI-based | String thickness multiplier. Scales T60_nyq: `T60_nyq_eff = T60_nyq / gauge`. Higher = more HF damping. Bass: 2-3, middle: 1.5, treble: 0.8-1.2 |
| `T60_fund` | float | 1.5-12 | MIDI-based | T60 of fundamental (seconds). Bass: 10-12s, treble: 1.5s |
| `T60_nyq` | float | 0.15-0.30 | MIDI-based | T60 at Nyquist (seconds, before gauge scaling). Controls spectral tilt |
| `n_strings` | int | 1-3 | MIDI-based | Unison strings. MIDI <= 27: 1, MIDI 28-48: 2, MIDI > 48: 3 |
| `detune_cents` | float | 0.3-2.5 | MIDI-based | Detuning between outer strings (cents). Creates beating + two-stage decay. Bass: 2-2.5, treble: 0.3-0.5 |

### Hammer

| Key | Type | Range | Default | Description |
|-----|------|-------|---------|-------------|
| `exc_x0` | float | 0.05-0.25 | 0.1429 (1/7) | Striking position as fraction of string length. Creates comb-filter notches at `k = 1/x0` multiples |
| `K_hardening` | float | 0-5.0 | 1.5 | Velocity-dependent stiffness scaling: `K_eff = K Ă— (1 + K_hardening Ă— vel_norm)`. Higher = brighter forte |
| `p_hardening` | float | 0-1.0 | 0.3 | Velocity-dependent exponent offset: `p_eff = p + p_hardening Ă— vel_norm`. Higher = sharper forte |
| `hammer_mass` | float | 0.1-3.0 | 1.0 | Hammer mass scale (1.0 = Chaigne default). Lower = lighter = shorter contact = brighter attack |
| `string_mass` | float | 0.1-3.0 | 1.0 | String mass scale (1.0 = Chaigne default). Higher = heavier = higher wave impedance |

Base K, p, Mh, Ms are interpolated from Chaigne & Askenfelt (1994)
anchor measurements. The scale parameters above multiply these base values.

### Dispersion

| Key | Type | Range | Default | Description |
|-----|------|-------|---------|-------------|
| `n_disp_stages` | int | 0-16 | from B | Allpass cascade stages. Auto: `min(16, int(B * N^2 * 0.5))`, minimum 3 or 0 |
| `disp_coeff` | float | -0.3-0 | -0.15 | Per-stage allpass coefficient |

### Output

| Key | Type | Range | Default | Description |
|-----|------|-------|---------|-------------|
| `output_scale` | float | 0.01-0.5 | 0.065 | Per-note output gain. Adjust for loudness calibration across keyboard |

## Loss filter (derived, not stored)

Computed at runtime from `T60_fund`, `T60_nyq`, and `gauge`:

```
N = sr / f0                          (samples per round-trip)
T60_nyq_eff = T60_nyq / gauge       (gauge scales HF damping)
g_dc  = 10^(-3 * N / (T60_fund * sr))
g_nyq = 10^(-3 * N / (T60_nyq_eff * sr))
pole  = (g_dc - g_nyq) / (g_dc + g_nyq)

H(z) = g_dc * (1 - pole) / (1 - pole * z^-1)
```

---

## Delay compensation (derived, not stored)

Each filter in the waveguide loop adds group delay that must be
subtracted from the delay line length to maintain correct tuning:

```
filter_delay = pole / (1 - pole^2)
disp_delay   = n_stages * (1 - a) / (1 + a)
N_comp       = sr/f0 - filter_delay - disp_delay
M            = N_comp / 2      (each rail = half period)
```

## Comparison with AdditiveSynthesisPianoCore JSON

| Aspect | Additive | Physical |
|--------|----------|----------|
| Notes | `m060_vel3` (per-velocity) | `m060` (velocity = hammer speed) |
| Partials | Explicit `partials[]` array | Continuous from Chaigne hammer |
| Decay | `tau1`, `tau2`, `a1` per partial | `T60_fund`, `T60_nyq` (filter-based) |
| Spectral EQ | `eq_biquads` (10-section IIR) | Loss filter + soundboard IR |
| Noise | `A_noise`, `noise_centroid_hz` | Hammer noise (bandpass white noise) |
| Inharmonicity | `B` + pre-computed `f_hz` | `B` + dispersion allpass cascade |
| Beating | `beat_hz` per partial | `detune_cents` between strings |
| Velocity layers | 8 layers (vel 0-7) | Continuous (Chaigne hammer physics) |
